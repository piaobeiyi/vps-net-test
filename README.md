测试客户端到服务器延迟

# 使用方法
1. 在服务器启动服务端
```bash
# IPv4 (同时启动 TCP 和 UDP 服务)
python3 latency_server.py

# 指定端口
python3 latency_server.py -p 9999

# IPv6
python3 latency_server.py -6

# IPv6 指定端口
python3 latency_server.py -6 -p 9999
```

服务端启动后会显示：

```

✓ TCP 服务已启动 (IPv4) - 端口 9999

服务端就绪，按 Ctrl+C 停止
============================================================
```
2. 在客户端运行测试
```bash
# 基本用法 (测试 ICMP → UDP → TCP)
python3 latency_client.py 192.168.1.100

# IPv6 测试
python3 latency_client.py 2001:db8::1

# 自定义参数
python3 latency_client.py 192.168.1.100 -p 9999 -c 50 --icmp-count 10 -s 128

# 跳过某些测试
python3 latency_client.py 192.168.1.100 --skip-icmp    # 只测试 UDP 和 TCP
python3 latency_client.py 192.168.1.100 --skip-udp     # 只测试 ICMP 和 TCP
python3 latency_client.py 192.168.1.100 --skip-tcp     # 只测试 ICMP 和 UDP
```

# 主要特性
✅ 服务端:

单一脚本同时启动 TCP 和 UDP 服务
使用相同端口 (通过不同协议)
多线程处理并发连接
支持 IPv4/IPv6
✅ 客户端:

自动按顺序测试: ICMP → UDP → TCP
自动检测 IPv4/IPv6
每个协议独立统计
最终生成对比报告
可选择跳过特定测试
✅ 测试顺序说明:

ICMP: 验证基础网络连通性
UDP: 测试无连接协议的延迟和丢包率
TCP: 测试面向连接协议的延迟（包括连接建立时间）
注意事项
ICMP 权限: ICMP ping 在某些系统上可能需要 root 权限
防火墙: 确保开放了指定的 TCP/UDP 端口
端口复用: TCP 和 UDP 可以使用相同端口号，因为它们是不同的传输层协议
